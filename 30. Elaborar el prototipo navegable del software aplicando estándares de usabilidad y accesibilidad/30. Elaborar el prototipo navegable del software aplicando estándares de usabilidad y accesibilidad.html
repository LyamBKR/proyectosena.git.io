📋 30.2 Sección 1: Taller (Prototipo funcional JavaScript)

Se desea elaborar un diagrama de funcionalidad, estableciendo colores y componentes a utilizar, basándose en el siguiente conjunto de funcionalidades relacionadas con el sistema de gestión de usuarios del videojuego:

Nota Importante:

Todas las funcionalidades deben desarrollarse según el diagrama de navegación del proyecto propuesto, el cual define el flujo de interacción del usuario dentro del sistema.

🔐

1. Pantalla de Autenticación

Se desea elaborar una pantalla que permita validar la autenticidad de un usuario (Según diagrama de navegación proyecto propuesto).

Funcionalidades requeridas:
  • Campo de usuario o email
  • Campo de contraseña
  • Validación de credenciales
  • Enlace para recuperación de contraseña
  • Integración con el sistema de usuarios del videojuego
📝

2. Pantalla de Registro de Usuario

Se desea elaborar una pantalla que permita ingresar nombres, apellidos, cédula, fecha de nacimiento de un usuario (Según diagrama de navegación proyecto propuesto).

Campos requeridos:
  • Nombres (obligatorio)
  • Apellidos (obligatorio)
  • Cédula/Número de identificación (obligatorio, único)
  • Fecha de nacimiento (obligatorio, validación de edad)
  • Email (opcional, para notificaciones)
  • Validación de datos antes de guardar
⚠️

3. Pantalla de Manejo de Errores

Se desea dibujar una pantalla en donde se informe de posibles errores a un usuario.

Tipos de errores a manejar:
  • Errores de autenticación (credenciales incorrectas)
  • Errores de validación (campos incompletos o inválidos)
  • Errores de conexión (problemas de red)
  • Errores de servidor (errores 500, 503)
  • Mensajes claros y amigables para el usuario
🎨

4. Propuesta de Paleta de Colores

Proponer la paleta de colores de los componentes anteriores utilizando como máximo la combinación de tres colores (Según diagrama de navegación proyecto propuesto).

Consideraciones para la paleta:
  • Máximo 3 colores principales
  • Contraste adecuado para accesibilidad
  • Coherencia con el estilo pixel art del videojuego
  • Colores que representen estados (éxito, error, advertencia)

🏠

Punto de Entrada

Pantalla inicial del sistema

Opciones disponibles:
  • Iniciar sesión
  • Registrarse como nuevo usuario
  • Acceso como invitado (si aplica)
🔐

Autenticación

Validación de credenciales de usuario

Flujos posibles:
  • Autenticación exitosa → Dashboard del juego
  • Autenticación fallida → Pantalla de error
  • Usuario no registrado → Pantalla de registro
📝

Registro

Creación de nueva cuenta de usuario

Flujos posibles:
  • Registro exitoso → Autenticación automática
  • Datos inválidos → Pantalla de error con detalles
  • Usuario ya existe → Pantalla de error específica
⚠️

Manejo de Errores

Sistema de notificación de errores

Opciones disponibles:
  • Reintentar la operación
  • Volver a la pantalla anterior
  • Contactar con soporte
  • Regresar al inicio

📚 30.3 Sección 2 – Taller Conceptos

Esta sección presenta los conceptos fundamentales relacionados con la calidad de software, usabilidad y estándares que deben aplicarse en el desarrollo del prototipo navegable.

📖 30.3.1. Definición de calidad de software

La calidad de software se define como el conjunto de características y propiedades que debe poseer un producto de software para satisfacer las necesidades y expectativas de los usuarios, cumpliendo con los requisitos funcionales y no funcionales establecidos.

Características principales de calidad:

  • Funcionalidad: Capacidad del software para proporcionar funciones que satisfagan necesidades explícitas e implícitas
  • Confiabilidad: Capacidad de mantener un nivel de rendimiento bajo condiciones establecidas
  • Usabilidad: Facilidad de uso y aprendizaje del software
  • Eficiencia: Capacidad de proporcionar un rendimiento adecuado en relación con los recursos utilizados
  • Mantenibilidad: Facilidad de modificar el software para corregir errores o agregar funcionalidades
  • Portabilidad: Capacidad del software para ser transferido de un entorno a otro

🎯 30.3.2. ¿Cuál es el concepto de usabilidad en software?

La usabilidad en software es la medida en que un producto puede ser usado por usuarios específicos para lograr objetivos específicos con efectividad, eficiencia y satisfacción en un contexto de uso determinado.

Componentes clave de la usabilidad (según ISO 9241):

Efectividad

Precisión y completitud con la que los usuarios alcanzan objetivos específicos

Eficiencia

Recursos utilizados en relación con la efectividad lograda

Satisfacción

Comodidad y aceptación del producto por parte de los usuarios

📋 30.3.3. ¿Cuáles son los estándares de calidad de software?

Los estándares de calidad de software son normas y especificaciones técnicas que establecen criterios, metodologías y mejores prácticas para garantizar la calidad en el desarrollo de software.

Estándares principales:

ISO/IEC 25010

Modelo de calidad de sistemas y productos de software. Define características de calidad como funcionalidad, rendimiento, compatibilidad, usabilidad, confiabilidad, seguridad, mantenibilidad y portabilidad.

ISO/IEC 9126

Estándar que define un modelo de calidad de software con características internas, externas y de calidad en uso.

ISO 9241

Ergonomía de la interacción humano-sistema. Define principios de diseño centrado en el usuario y criterios de usabilidad.

WCAG 2.1

Pautas de Accesibilidad para el Contenido Web. Establece estándares para hacer el contenido web accesible para personas con discapacidades.

🏠 Botón Home - Implementación y CSS

Descripción del Componente

El botón "Home" permite a los usuarios regresar rápidamente al índice principal del proyecto desde el prototipo navegable. Este componente está diseñado siguiendo los principios de accesibilidad y usabilidad, utilizando CSS inline para mantener la consistencia visual con el resto del prototipo.

📝 Código HTML Completo del Botón Home

<a href="/index.html" 
   class="btn-text" 
   style="background: #1976D2; 
          color: white; 
          border: 2px solid #1565C0; 
          text-decoration: none; 
          display: inline-flex; 
          align-items: center; 
          justify-content: center; 
          padding: 12px 24px; 
          border-radius: 8px; 
          font-weight: 600; 
          transition: all 0.3s ease; 
          box-shadow: 0 4px 12px rgba(25, 118, 210, 0.3);" 
   onmouseover="this.style.background='#1565C0'; 
                this.style.transform='translateY(-2px)'; 
                this.style.boxShadow='0 6px 16px rgba(25, 118, 210, 0.4)';" 
   onmouseout="this.style.background='#1976D2'; 
               this.style.transform='translateY(0)'; 
               this.style.boxShadow='0 4px 12px rgba(25, 118, 210, 0.3)';">
    🏠 Home
</a>

🔍 Explicación Línea por Línea del Código

1. Etiqueta de Apertura y Atributos Principales
<a href="/index.html" class="btn-text">
  • <a> - Elemento de anclaje HTML5, semánticamente correcto para navegación
  • href="/index.html" - Ruta absoluta al índice principal del proyecto. La barra inicial (/) indica la raíz del sitio
  • class="btn-text" - Clase CSS que puede tener estilos base definidos en el archivo styles.css
2. Estilos CSS Inline - Propiedades de Color y Fondo
style="background: #1976D2; 
       color: white; 
       border: 2px solid #1565C0;"
  • background: #1976D2 - Fondo azul Material Design Blue 700. Este color es el estado normal del botón
  • color: white - Color del texto en blanco (#FFFFFF) para máximo contraste sobre el fondo azul
  • border: 2px solid #1565C0 - Borde sólido de 2 píxeles en azul más oscuro (Blue 800) para definir mejor el botón
3. Estilos CSS - Layout y Display
text-decoration: none; 
display: inline-flex; 
align-items: center; 
justify-content: center;
  • text-decoration: none - Elimina el subrayado predeterminado de los enlaces
  • display: inline-flex - Combina inline (permite estar en línea con otros elementos) y flexbox (permite alineación flexible)
  • align-items: center - Alinea verticalmente el contenido (icono y texto) al centro del contenedor flex
  • justify-content: center - Alinea horizontalmente el contenido al centro del contenedor flex
4. Estilos CSS - Espaciado y Dimensiones
padding: 12px 24px; 
border-radius: 8px; 
font-weight: 600;
  • padding: 12px 24px - Espaciado interno: 12px arriba/abajo, 24px izquierda/derecha. Esto crea un área de clic cómoda y cumple con estándares de accesibilidad (mínimo 44x44px)
  • border-radius: 8px - Bordes redondeados de 8 píxeles para un diseño moderno y suave
  • font-weight: 600 - Peso de fuente semi-negrita (600) para mejor legibilidad y jerarquía visual
5. Estilos CSS - Efectos y Transiciones
transition: all 0.3s ease; 
box-shadow: 0 4px 12px rgba(25, 118, 210, 0.3);
  • transition: all 0.3s ease - Transición suave de 0.3 segundos para todas las propiedades CSS que cambien. "ease" crea una curva de animación natural (lento-rápido-lento)
  • box-shadow: 0 4px 12px rgba(25, 118, 210, 0.3) - Sombra del botón:
    • 0 - Desplazamiento horizontal (sin desplazamiento)
    • 4px - Desplazamiento vertical hacia abajo
    • 12px - Radio de difuminado (blur)
    • rgba(25, 118, 210, 0.3) - Color azul con 30% de opacidad
6. Evento onmouseover - Estado Hover
onmouseover="this.style.background='#1565C0'; 
                this.style.transform='translateY(-2px)'; 
                this.style.boxShadow='0 6px 16px rgba(25, 118, 210, 0.4)';"
  • onmouseover - Evento JavaScript que se ejecuta cuando el cursor del mouse entra en el área del botón
  • this.style.background='#1565C0' - Cambia el fondo a un azul más oscuro (Blue 800) para indicar interactividad
  • this.style.transform='translateY(-2px)' - Mueve el botón 2 píxeles hacia arriba, creando efecto de "elevación"
  • this.style.boxShadow='0 6px 16px rgba(25, 118, 210, 0.4)' - Aumenta la sombra (6px desplazamiento, 16px blur, 40% opacidad) para efecto 3D más pronunciado
7. Evento onmouseout - Restaurar Estado Normal
onmouseout="this.style.background='#1976D2'; 
               this.style.transform='translateY(0)'; 
               this.style.boxShadow='0 4px 12px rgba(25, 118, 210, 0.3)';"
  • onmouseout - Evento JavaScript que se ejecuta cuando el cursor sale del área del botón
  • this.style.background='#1976D2' - Restaura el color de fondo original (Blue 700)
  • this.style.transform='translateY(0)' - Devuelve el botón a su posición original (sin desplazamiento vertical)
  • this.style.boxShadow='0 4px 12px rgba(25, 118, 210, 0.3)' - Restaura la sombra original del estado normal
8. Contenido y Cierre de la Etiqueta
🏠 Home
</a>
  • 🏠 - Emoji de casa (Unicode U+1F3E0) que mejora la identificación visual del botón y proporciona contexto inmediato
  • Home - Texto descriptivo que indica claramente la función del botón
  • </a> - Cierre de la etiqueta de anclaje, finalizando el elemento

♿ Características de Accesibilidad

  • Feedback visual: Los estados hover proporcionan retroalimentación clara al usuario
  • Navegación por teclado: Como elemento <a>, es accesible mediante Tab y Enter
  • Semántica HTML: El uso de enlace (<a>) es semánticamente correcto para navegación
